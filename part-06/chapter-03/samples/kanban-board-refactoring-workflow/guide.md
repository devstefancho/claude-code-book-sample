리팩터링을 위한 클로드 코드 기능들을 개발한다.

1. test 커맨드 개발

- test 코드를 돌리는 커스텀 슬래시 커맨드를 개발한다.
- test가 실패하는 케이스에서는 test를 fix하는 subagent를 실행하여 test를 고친다.

2. code-review 스킬 개발

- 프로젝트 전반을 확인하여 코드 리뷰를 진행한다.
- 코드 리뷰는 subagent를 최대한 활용한다.
- 장기적 유지보수 관점에서 문제가 될 수 있는 기능들을 선별하여 사용자에게 리팩토링을 제안한다.

3. refactor 스킬 개발

- 새로운 git worktree와 브랜치를 생성한다.
- code-review에서 사용자가 리팩토링을 요청한 내용들을 subagent를 사용해서 docs/{현재 브랜치명}/refactor.md 에 작성한다.
- 그리고 메인 에이전트에서 리팩토링을 진행한다.
- 리팩토링이 끝나면 test 커맨드를 실행한다.
