## 단축키 알아보기

`[?]`를 눌러서 사용할 수 있는 단축키를 확인할 수 있습니다.

프롬프트 첫 단어의 특수문자에 따른 기능:
- `[!]` - Bash 모드
- `[/]` - 슬래시 커맨드
- `[@]` - 파일 멘션
- `[&]` - 특별 기능

## Bash 모드

클로드 코드에서 `[!]`를 입력하면 명령어를 입력할 수 있는 모드가 됩니다.

### bash mode 사용방법

1. `[!]`를 입력합니다.
2. 터미널 명령어를 입력하고 `[Enter]`를 누릅니다.
3. 명령어 실행 후 자동으로 일반 입력모드로 돌아옵니다.
4. `[Backspace]`로 bash mode를 빠져나갈 수 있습니다.

### bash mode의 활용방안

권한 설정으로 클로드 코드가 읽지 못하는 파일을 사용자가 직접 보여줄 수 있습니다.

#### .env 파일 생성

```shell
echo "secret-key" > .env
```

#### .claude/settings.json 권한 설정

```json
{
  "permissions": {
    "deny": ["Read(.env)"]
  }
}
```

## 슬래시 커맨드 사용하기

`[/]`를 입력하면 사용가능한 슬래시 커맨드가 나옵니다.

### /export 커맨드 사용 예시

1. `/export`를 입력하고 `[Enter]`
2. 선택창에서 "Save to file" 선택
3. 파일명 입력 후 저장

또는 `/export hello.txt`처럼 인자를 직접 넣어서 실행할 수 있습니다.

## 파일멘션하기

`[@]`를 입력하면 파일 또는 디렉토리 경로가 자동으로 검색됩니다.

```shell
touch hello.sh hello.txt hello.md
```

### gitignore 파일을 멘션하는 방법

`.gitignore`에 추가된 파일도 검색하려면:

```json
{
  "respectGitignore": false
}
```

## 모델 선택하기

클로드 모델 종류:
1. **Opus**: 가장 똑똑한 모델. 작업계획, 복잡한 로직, 리팩터링에 적합
2. **Sonnet**: 중간 수준. 어렵지 않은 코드 작업에 적합
3. **Haiku**: 가장 경량. 단순한 일이나 문서작업에 적합

### /model 커맨드로 모델 선택

`/model` 명령어를 입력하고 원하는 모델을 선택합니다.

### 기본 모델 설정

```json
{
  "model": "Opus"
}
```

### 프롬프트 입력중에 모델 변경

`[Alt] + [p]`를 눌러 모델 선택창을 띄울 수 있습니다.

## 프롬프트 입력하기

### 프롬프트 비우기

- `[Esc] + [Esc]` 또는 `[Ctrl] + [c]`로 입력창을 비웁니다.

### 프롬프트 히스토리 탐색하기

- 위/아래 방향키로 이전 프롬프트 탐색
- `[Ctrl] + [r]`로 프롬프트 검색

### 여러줄 입력하기

- `[Alt] + [Enter]`
- `[\]` + `[Enter]`
- `[Shift] + [Enter]` (터미널에 따라 `/terminal-setup` 필요)

### 프롬프트를 외부 에디터에서 작성하기

#### 에디터 설정값 확인

```shell
echo $EDITOR # nvim
```

#### VSCode를 외부 에디터로 설정

```json
{
  "env": {
    "EDITOR": "code --wait"
  }
}
```

`[Ctrl] + [g]`를 입력하면 외부 에디터가 실행됩니다.

### 이미지 붙여넣기

이미지를 참조하는 3가지 방법:

1. **클립보드 붙여넣기**: `[Ctrl] + [v]`
2. **드래그 앤 드롭**: 파일 탐색기에서 이미지를 드래그
3. **멘션**: `[@]`로 이미지 파일 멘션

#### 프롬프트 예시

```
[나]: @mascot.png 이미지에 대해서 간략히 한 줄로 설명해줘

[AI]: 갈색 픽셀 아트 스타일의 귀여운 문어(또는 오징어) 캐릭터 마스코트입니다.
```

### 프롬프트 임시 저장하기

`[Ctrl] + [s]`를 누르면 현재 프롬프트가 임시 저장(Stash)되고, 다음 프롬프트 제출 후 자동으로 복구됩니다.

### Vim 모드

`/vim` 커맨드 또는 `/config`에서 Vim 모드를 켜거나 끌 수 있습니다.

## 클로드 코드의 Thinking Mode

복잡한 문제를 분석하고 해결 전략을 세우는 추론(Reasoning) 과정을 보여주는 모드입니다.

- `[Alt] + [t]`로 켜거나 끌 수 있습니다.
- `/config`에서도 설정 가능합니다.

## 메시지 추가정보 확인하기

- `[Ctrl] + [o]`로 상세 모드(Verbose Mode)를 활성화합니다.
- 도구 사용 세부정보, 실행 과정, Thinking 내용을 확인할 수 있습니다.

### verbose 플래그로 실행

```shell
claude --verbose
```
