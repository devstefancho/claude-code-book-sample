# Chapter 03. SDD 전체 내용 되짚어보기

> 프롬프트: [prompts.md](./prompts.md) | 코드: [code.md](./code.md)

SDD(Spec-Driven Development)를 통해 프론트엔드부터 데이터베이스 연동까지 개발하는 전체 과정을 정리합니다.

## 프론트엔드부터 데이터베이스 개발까지 되짚어보기

### 전체 과정 요약

1. kanban-board 프로젝트 생성
2. frontend-setup 브랜치 생성
3. /sdd:spec 커맨드 생성
4. 프론트엔드 개발을 위한 스펙파일 생성
5. /sdd:plan 커맨드 생성
6. 메인 기술 스택 정하고 커맨드 실행
7. /sdd:tasks 커맨드 생성
8. 태스크 커맨드 실행 (Arguments 없이)
9. /sdd:implement 커맨드 생성
10. `--all` Arguments로 한번에 구현 진행
11. db-integration 브랜치 생성
12. Supabase 가입 및 프로젝트 생성, MCP 설정
13. 스펙 커맨드 실행 (유저 관련 내용 포함)
14. 플랜 커맨드 실행 (supabase, supabase mcp 명시)
15. 태스크 커맨드 실행
16. 구현 커맨드로 완료

## 문서 기반 개발의 강력함을 이해하고, 더 멀리 나아가기

SDD가 항상 추천되는 방식은 아닙니다:
- **쉽고 작은 기능**: 단계를 나누지 않아도 됨
- **복잡한 기능**: 더 많은 단계가 필요할 수 있음

### 확장 가능한 단계들

- **테스트 단계**: 테스트 문서 생성
- **보안점검 단계**: 보안점검 문서 생성

**핵심**: 필요한 만큼 문서를 만들어 안정적인 스펙으로 개발하는 것이 가장 빠른 지름길입니다.
