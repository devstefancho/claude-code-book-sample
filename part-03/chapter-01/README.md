# Chapter 01. 데모는 가깝고 실제 서비스는 멀게 느껴지는 현실

> 프롬프트: [prompts.md](./prompts.md) | 코드: [code.md](./code.md)

클로드 코드를 사용해도 완성도 있는 서비스를 만드는 것은 쉽지 않습니다. 한두 번 기능개발에 성공하는 것과 서비스 개발 전체를 성공하는 것은 차원이 다른 문제입니다.

## 작성자와 편집자

- **과거**: 코드 작성자(Writer)의 역할이 주요 역량
- **현재**: 코드 편집자(Editor)의 역할이 주요 역량

### 편집자의 역할

- 서비스를 확장하는데 지속 가능한 구조인가 판단
- 코드 스멜(Code Smell)을 맡는 감각 기르기
- 리팩터링 패턴의 이름을 익히고 예시를 통해 원칙 이해

## 적절한 개입으로 서비스의 사령관이 되자

### HITL (Human In The Loop)

AI 시스템이 작업을 수행하는 과정에 인간이 필수로 개입하여 결과물을 검증, 수정, 승인하는 프로세스입니다.

HITL을 최소화하려면:
- 개발 초기에 환경을 잘 만들어 두기
- 코드베이스를 일관되고 안전한 코드들로 구축

### 코드 베이스 구축 원칙

- 오버 엔지니어링을 피하고 이펙티브 엔지니어링 하기
- 기능이 추가되면서 레거시가 되는 상황 대비
- 서비스의 목적지와 방향성은 개발자가 결정
- 리팩터링 패턴들과 장단점에 대한 이해 필요
