## 스킬 개선 요청

```
[나]: 컴포넌트 디자인 리뷰 스킬을 개선해 줘. 단일 책임 원칙을 너무 엄격하게 적용해서, 오히려 Props Drilling이 심해지는 부작용이 종종 발생했어. 컴포넌트를 분리하라고 제안하더라도 Props 전달이 3단계를 넘지 않도록 예외 조건을 추가해 줘.

정식 Eval 루프를 사용해서 기존 버전과 새 버전을 비교해 줘. 여러 컴포넌트를 한 번에 검토하면 리소스 병목이 올 수 있으니, 테스트 케이스는 반드시 순차적으로 하나씩 실행해 줘.

[AI]: 기존 스킬을 먼저 확인하겠습니다.

현재 스킬을 확인했습니다. 문제가 명확하네요. SRP 검사에서 컴포넌트 분리를 제안하면서 Props Drilling 깊이를 고려하지 않고 있습니다.

개선 방향
- SRP 체크리스트에 Props Drilling 가드레일 추가
- 원칙 간 충돌 해결 가이드 추가

작업을 시작하겠습니다.
```

## 스킬 개선 테스트 진행

```
[나]: 테스트를 진행해줘

[AI]: 순차적으로 테스트를 실행합니다. 먼저 iteration-1 디렉토리와 eval metadata를 준비하겠습니다.

이제 Eval 0 단계인 AdminDashboard에서 old_skill과 with_skill을 순차 실행합니다.
Eval 0를 완료했습니다. Eval 1 단계인 CheckoutFlow 실행합니다.
Eval 1를 완료했습니다. Eval 2 단계인 ProjectBoard 실행합니다.

Eval 0, Eval 1, Eval 2 이렇게 총 3가지의 평가가 순차적으로 진행됩니다.
```
