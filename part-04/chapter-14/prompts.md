## 스킬 개선 요청 (정식 Eval 루프 + 순차 실행)

```
[나]: 컴포넌트 디자인 리뷰 스킬을 개선해 줘. 단일 책임 원칙을 너무 엄격하게 적용해서, 오히려 Props Drilling이 심해지는 부작용이 종종 발생했어. 컴포넌트를 분리하라고 제안하더라도 Props 전달이 3단계를 넘지 않도록 예외 조건을 추가해 줘. 정식 Eval 루프를 사용해서 기존 버전과 새 버전을 비교해 줘. 여러 컴포넌트를 한 번에 검토하면 리소스 병목이 올 수 있으니, 테스트 케이스는 반드시 순차적으로 하나씩 실행해 줘.
```

```

```


## 스킬 개선 프롬프트 작성 공식

skill-creator로 스킬을 개선할 때는 다음 3가지를 프롬프트에 반드시 포함합니다.

1. **정확한 증상과 제약 조건**: "Props 전달이 3단계를 넘지 않아야 해"처럼 타협할 수 없는 구체적인 조건을 명시합니다.
2. **평가 루프 강제 호출**: "그냥 수정하지 말고 정식 Eval 루프를 사용해 줘"라고 해야 전후 비교 벤치마크 워크스페이스가 정상 작동합니다.
3. **순차 실행 지시**: 대규모 코드 리뷰처럼 시간이 오래 걸리는 작업은 "한 번에 돌리지 말고 순차적으로 실행해 줘"라고 해야 CPU/IO 병목으로 클로드 코드가 멈추는 것을 방지할 수 있습니다.
